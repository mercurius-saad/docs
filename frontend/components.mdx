## Styling approach

No external design system. Custom CSS with:

- **CSS variables** for colors (see [Overview](/frontend/overview))
- **Utility classes** in `defaultStyle.css` (flex, grid, margin/padding, font)
- **Scoped styles** in each `.vue` file's `<style scoped>` section
- **Bootstrap 4.3** for grid and responsive basics
- **SCSS/Sass** preprocessor
- **Custom fonts**: Inter (8 weights), Hanson Bold, Autography, Inconsolata, DepartureMono

## Registration flow components

The registration is a 6-step wizard orchestrated by `views/Register.vue`:

<Steps>
<Step title="Device.vue">
iOS or Android selection. Android users redirected to Apply page.
</Step>
<Step title="Username.vue">
Username input (5-20 chars, starts with letter, alphanumeric). Real-time availability check via API. Country code selector (`NumberCode.vue`) + phone number input.
</Step>
<Step title="Identity.vue">
First name, last name, date of birth (day/month/year as separate inputs). Age validation (must be 18+). Auto-focus progression through fields.
</Step>
<Step title="Email.vue">
Email with real-time uniqueness check. Password with strength validation (min 8 chars, 1 uppercase, 1 special). Show/hide toggle. `EmailVerify.vue` confirmation popup.
</Step>
<Step title="Address.vue">
Country dropdown â†’ postcode input â†’ real-time address search â†’ `AddressPicker.vue` for manual entry. Lat/lng captured.
</Step>
<Step title="Payment.vue">
Card number (12-19 digits), expiry (MM/YY with auto-formatting), CVC (3-4 digits). Input masking, auto-progression between fields. Test card detection.
</Step>
</Steps>

Step completion emits `confirm-step(stepNumber, data)` to the parent `Register.vue`.

## Form handling pattern

No form library. All forms use:

```javascript
// Local state
data() {
  return {
    email: '',
    error: '',
    validInput: false
  }
}

// Validation in methods
methods: {
  validate() {
    return this.email.length > 0 && this.validInput;
  }
}

// Template binding
<input v-model="email" @keyup.enter="submit" />
<button :disabled="!validate()" :class="{ disabledInput: !validate() }">Submit</button>
<span class="color-red" v-if="error">{{ error }}</span>
```
